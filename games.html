AP Research game

December 2020 - May 2021

While I don't have a playable version of the game anymore or the code, I do have some evidence of its existance in my AP Research paper (linked in title). This game was an online multiplayer slide puzzle game that I used to test how different types of leaderboards affected performance in a cognitive task. I got the code for the game online, then added upon it by integrating Google Sheets using an API to store all player data during and after the game. Students then played it during a math class with permission from the teacher. In some classes, the game crashed for people, knocking them out of the game one at a time, which was honestly kinda funny (in retrospect, I think it had something to do with a quota for API read and write calls). Still, I was quite proud that I got a semi-working game with such ambition, and I think this was a key moment that solidified my love of making games (although I wouldn't explore it further for another year).

Swordfight

April 2018?

The very first game I ever made. While I have been unable to find any footage of or code for this game, I can provide you with its story.

I was in ninth grade, taking Ms. Lint's Intro to Programming, and in the second half of the year, we were taking CMU's CS Academy course to learn programming (after using CMU's Alice during the first half). The course was self-paced, but I found programming so fun that I tore through the class at lightning speed. Near the end of the year, Ms. Lint told us about a contest CMU CS Academy was holding between all schools who participated in the course. I decided to enter the toughest division which involved creating a game of some sort. There was no real game engine, but I quickly made a local multiplayer game with rudimentary graphics in about a class period or two, which featured two characters, each consisting of a circle (representing a head) overtop a rectangle (representing shoulders), in a top-down view. Each player could move around using WASD or arrow keys, as well as attack using a button, which would make a small, skinny rectangle (representing a sword) appear. To win, you needed to strike the opponent with your sword. I didn't think it was that good, but it was kinda fun to play, and I actually ended up winning 2nd place at the event (losing to someone who made a Perry the Platypus platformer, like how?), which earned me a Kindle Fire 7 as a reward. Even if I put in a ton more effort into this game or another, I certainly wouldn't have won, so I am definitely not disappointed about 2nd place. Unfortunately, I had no use for the Kindle (it even lagged running 2048), so I don't know where it is today.

While the game totally sucked, it was still charming, creative, and fun, which honestly sums up my current games today. Because it was my first programmed video game, I will always hold the game and the experience dear to my heart. I will never forget the suspenseful excitement watching people test, play, and enjoy a game that I made :)
